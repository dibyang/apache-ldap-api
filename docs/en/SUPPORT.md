# Support

[中文版本](../../SUPPORT.md)

## Getting Help

Use GitHub Issues for questions and problem reports. Search existing issues before filing a new one to avoid duplicate discussions.

For reproducible bugs, use the GitHub bug report form and follow `../bug-reporting.md` when preparing a minimal reproducer. For security issues, read `../../SECURITY.md` and do not disclose unfixed vulnerability details in public issues.

## Include This Information

- Project version or commit hash.
- JDK version and operating system.
- Gradle command that was run.
- Expected result and actual result.
- Minimal reproducer, configuration, or test case.
- Complete stack trace or the most relevant build log excerpt.

## Good Issue Topics

- Reproducible bugs.
- Missing or unclear documentation.
- Build, test, or release workflow problems.
- Compatibility issues.
- Well-scoped feature requests.

## Not for Public Issues

- Detailed exploit steps for unfixed security vulnerabilities.
- Private credentials, internal addresses, production data, or personal information.
- General LDAP operations questions unrelated to this project.
